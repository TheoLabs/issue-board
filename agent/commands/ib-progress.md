너는 이 프로젝트의 개발자다. 지금까지의 작업을 근거로 **이슈의 완료 조건 체크박스와
상태를 실제 진행도에 맞게 동기화**한다. 대시보드에서 사람이 체크하는 것과 같은 편집을
대신 하는 것이다.

## 대상

$ARGUMENTS

(비어 있으면: 최근 대화/작업 맥락에서 지금 진행 중인 이슈를 추정하고, 애매하면 물어라.)

## 진행 순서

### 1) 이슈 찾기 · 읽기

- 이슈 id가 있으면 `get_issue(issueId)`로 읽는다.
- id가 없으면 `get_project_context(repoPath=<cwd 절대경로>)`(또는 `list_issues`)로 목록을
  받아 대상 이슈를 찾는다 (제목/맥락 매칭). 없으면 물어라.

### 2) 완료 조건 대조

이슈 본문의 "완료 조건" 체크리스트(`- [ ]` / `- [x]`)를 하나씩 본다. **실제 코드/작업
결과를 근거로** 각 항목이 끝났는지 판단한다.

- 끝난 항목: `- [ ]` → `- [x]`
- 아직인 항목: 그대로 둔다 (근거 없이 체크하지 마라)
- **추측으로 체크하지 말 것.** 실제로 구현/검증된 것만.

### 3) 저장

- 본문을 수정했으면 `update_issue(issueId, body=<수정본 전체>)`로 저장한다.
  **완료 조건 체크박스 외 다른 본문은 건드리지 마라.**
- 진행도에 맞춰 상태도 조정:
  - 일부라도 착수됐는데 `todo`면 → `update_issue_status(issueId, "in_progress")`
  - 완료 조건이 **전부 체크**되면 → 모두 `- [x]` 확인 후 `update_issue_status(issueId, "done")`
  - 막혔으면 → `"blocked"`

### 4) 보고

체크한 항목 / 남은 항목 / 최종 상태를 요약한다.

## 주의

- 근거 없는 체크·상태 변경 금지. **실제 작업 결과에만** 근거한다.
- 완료 조건 체크리스트와 상태 외의 것은 수정하지 마라 (제목·우선순위 등은 손대지 않음).
