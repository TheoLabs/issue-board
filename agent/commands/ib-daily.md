description: 오늘(또는 지정일) 이슈보드에서 한 작업을 일일 업무 요약으로 정리해 구글 드라이브에 올린다
argument-hint: <날짜(YYYY-MM-DD, 생략 시 오늘)> 또는 비워두기
---

너는 이 프로젝트의 개발자다. 이슈보드에 쌓인 **오늘의 변경 이력(활동 로그)** 을 읽어
**정규화된 일일 업무 요약**을 만들고, **구글 드라이브에 업로드**한다.

요약의 원천 데이터는 이슈보드 서버의 활동 로그(ActivityLog)다. 추측하지 말고 **활동
로그에 실제로 기록된 것만** 요약한다.

## 대상 날짜

$ARGUMENTS

(비어 있으면: 오늘 — 서버 타임존(기본 Asia/Seoul) 기준.)

## 진행 순서

### 1) 프로젝트 · 활동 수집

- `get_project_context(repoPath=<cwd 절대경로>)` 로 현재 프로젝트를 매칭한다.
  - 매칭 실패면 사용자에게 어떤 프로젝트인지 물어라 (`list_projects`).
- `get_daily_activity(projectId, date=<대상 날짜|생략>, timezone="Asia/Seoul")` 로 하루치
  활동을 받는다. 응답 필드:
  - `total`, `byEntity`, `byAction`, `bySource`
  - `activities[]`: `entityType`(project|plan|issue|domain|wireframe|design) ·
    `action`(created|updated|status_changed|snapshot|linked|deleted) · `title` ·
    `changes`(예: `{status:{from,to}}`) · `source`(user|agent) · `createdAt`
- `total`이 0이면: "오늘 기록된 변경이 없습니다"만 알리고 **업로드하지 않는다**(빈 문서
  생성 금지). 사용자가 원하면 그때 만든다.

### 2) 정규화 템플릿으로 요약 작성 (하이브리드 · 엔티티별)

아래 **고정 구조**를 그대로 따른다. 상단 TL;DR만 네가 서술하고, 나머지는 활동 로그를
엔티티별로 기계적으로 정리한다. 데이터가 없는 섹션은 통째로 생략한다.

```markdown
# 📋 일일 업무 요약 — {프로젝트명}
{YYYY-MM-DD} ({요일}) · 자동 생성 by ib-daily

## TL;DR
{오늘 한 일을 1~2문장으로 서술. 무엇을 진척시켰는지 중심. 과장·추측 금지.}
기획 {n} · 도메인 {n} · 이슈 {n} · 와이어프레임 {n} · 디자인 {n} — 총 {total}건

## 🗂 기획
- {신규|수정|스냅샷|삭제}: "{title}"{ status 변경 시 ` (from → to)`}

## 🧩 도메인
- {추가|수정|삭제}: "{title}"

## ✅ 이슈
**완료** {상태변경으로 done이 된 것 + 오늘 done으로 만든 것}
- "{title}"
**진행 중** {in_progress로 전이}
- "{title}" (todo → in_progress)
**신규**
- "{title}"
**기타 변경** {수정/연동/삭제}
- "{title}"

## 🖼 와이어프레임 / 🎨 디자인
- {추가|수정|삭제}: "{title}"

## ⏭ 내일 이어서
- {오늘 in_progress로 남아 있는 이슈 제목들. 없으면 이 섹션 생략}

---
출처: issue-board · 주체 표기 🤖=Claude(MCP), (없음)=웹 사용자 · 생성 {ISO 시각}
```

작성 규칙:
- **엔티티별**로 묶는다(기획 → 도메인 → 이슈 → 와이어프레임/디자인 순).
- 이슈는 액션이 아니라 **결과 상태 중심**으로 소분류(완료/진행 중/신규/기타).
- `source=agent`(Claude/MCP)로 생긴 항목 끝에는 🤖 를 붙여 사람 작업과 구분한다.
- 같은 대상이 여러 번 바뀌었으면 **마지막 상태로 합쳐** 한 줄로 쓴다.
- 없는 섹션·빈 소분류는 출력하지 않는다.

### 3) 구글 드라이브 업로드

구글 드라이브 MCP 도구로 올린다. **폴더 구조 = 루트 / 프로젝트, 파일 = 날짜별 문서.**

1. 루트 폴더 `이슈보드 일일요약` 을 검색해 없으면 만든다.
2. 그 아래 프로젝트 폴더 `{프로젝트명}` 을 검색해 없으면 만든다.
3. 그 폴더에 문서를 만든다:
   - 파일명: `{YYYY-MM-DD} {프로젝트명} 일일요약`
   - 내용: 2)에서 만든 마크다운 (Google Docs 문서로 생성)
   - **같은 날짜 문서가 이미 있으면 새로 만들지 말고 그 내용을 갱신**한다(덮어쓰기).
4. 업로드가 끝나면 **문서 링크(webViewLink)** 를 사용자에게 보고한다.

> 구글 드라이브 연동이 없거나 실패하면: 요약 본문을 대화에 그대로 출력하고, 업로드는
> 건너뛰었다고 알려라. (요약 자체는 항상 보여준다.)

### 4) 보고

- 만든 요약의 핵심(총 건수 + TL;DR)과 **구글 드라이브 문서 링크**를 요약해 알린다.

## 매일 자동 실행 (스케줄)

무인으로 매일 돌리려면 이 커맨드를 스케줄에 건다. 예: 매일 저녁 7시(KST)

```
/schedule 매일 19:00 KST 에 /ib-daily 를 이 프로젝트에서 실행해 구글 드라이브에 올려줘
```

스케줄 실행 환경에는 issue-board MCP와 구글 드라이브 연동이 모두 붙어 있어야 한다.

## 주의

- **활동 로그에 있는 것만** 요약한다. 코드·대화 기억으로 항목을 지어내지 마라.
- 활동 로그는 이 기능 도입 시점부터 쌓인다. 그 이전 날짜는 데이터가 없을 수 있다.
- 업로드는 되돌리기 번거로우니, 같은 날짜 문서는 **새로 만들지 말고 갱신**한다.
