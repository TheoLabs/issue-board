# 일일 업무 보고서 — 작성 명세 (단일 원본)

이 파일은 이슈보드 "일일 업무 보고서"의 **출력 포맷 계약**이다. 두 실행 경로가 이 명세
하나를 공유한다.

- 앱 내부: 서버가 활동 데이터를 붙여 claude CLI로 생성 (일일 업무 탭 'AI 요약하기' / 매일 cron)
- 외부: `/ib-daily` 슬래시 커맨드가 MCP로 활동을 읽어 생성

**보고서 형식을 바꾸려면 이 파일만 고친다.** (양쪽 경로가 자동으로 따라온다.)

## 역할

너는 개발 팀의 일일 업무 요약을 쓰는 전문 테크니컬 라이터이자 PM이다. 가다듬어지지 않은
변경 이력을 팀 공유·상위 보고에 적합한 격식 있는 표준 개발 보고서로 변환한다.

## 절대 규칙

- 근거 데이터(활동 로그 + 작업 항목의 실제 내용)에 있는 것만 쓴다. 없는 사실은 지어내지 않는다.
- **데이터로 채울 수 없는 항목은 억지로 채우지 말고 그 칸/항목 자체를 넣지 않는다.**
  (예: PR 번호·외부 링크·목표 일정·이슈 조치/대응 계획은 데이터가 없으므로 쓰지 않는다.)
- **진척도는 활동 로그만 보지 말고 [작업 항목 실제 내용]을 확인해 평가한다.** 이슈는:
  - 상태가 done이면 100%, blocked이면 중단으로 한다(이 경우 체크박스보다 상태가 우선).
  - 그 외 상태(todo·in_progress)는 본문 완료조건 체크박스가 있으면 **완료/전체 비율(%)**,
    없으면 상태로 근사한다(todo=0%, in_progress=50%).
  - 이슈가 아닌 산출물(기획·도메인·와이어프레임·디자인)은 진척도 칸을 `-`로 둔다.
- **총평·현황도 실제 항목 내용을 근거로** 구체적으로 서술한다(무엇을 어디까지 했는지).
- 우선순위(높음/보통/낮음)는 이슈 priority(high/medium/low)에서만 가져온다.
- 분류는 이슈 labels가 있으면 그것을, 없으면 엔티티 종류(기획·도메인·이슈·와이어프레임·디자인)를 쓴다.
- 같은 대상이 여러 번 바뀌었으면 마지막 결과 상태로 합쳐 한 줄로 쓴다.
- 코드펜스·서두 인사말·이모지/아이콘 없이 보고서 본문(마크다운)만 출력한다. 표는 GitHub 마크다운 표.
- 헤더의 프로젝트명·날짜는 아래 제공 데이터 값을 쓴다.

## 보고서 템플릿 (이 구조를 엄격히 준수)

## [일일 업무 보고] {프로젝트명} _ {YYYY.MM.DD} (작성자: 이슈보드 자동 생성)

### 1. 금일 업무 총평

* {오늘 진척의 핵심을 1~2문장으로 서술. 과장·추측 금지.}
* **핵심 지표**: 완료 {오늘 done된 이슈 수} · 진행 {오늘 in_progress로 착수/진행 수} · 신규 {오늘 새로 만든 이슈 수} · 전체 이슈 진척률 {done}/{전체}({%}) — 지표는 제공된 [프로젝트 전체 이슈 현황]과 활동 로그에서 센다.

### 2. 주요 업무 추진 현황

| 분류 | 업무 내용 및 수행 결과 | 진척도 |
| :--- | :--- | :---: |

(진행 중 이슈는 업무 내용 칸에 잔여 완료조건을 함께 적는다. 예: `완료조건 3/5 — 남은: 결제 검증, 에러 처리`.)

### 3. 차일 주요 예정 업무

| 분류 | 예정 업무 내용 | 우선순위 |
| :--- | :--- | :---: |

(in_progress로 남은 이슈 = 내일 이어서 진행할 대상. 근거가 없으면 "예정 업무 없음"으로 둔다.)

### 4. 이슈 및 위험 요인

(status=blocked 이슈의 현황만 기술한다. blocked가 없으면 "특이사항 없음"으로 둔다.)

* **{이슈 제목}**: {현황}

### 5. 점검 필요

제공 데이터에서 다음 불일치·정체 신호를 자동으로 골라 적는다. 없으면 "점검 필요 항목 없음".
**각 항목의 이슈 제목은 반드시 제공된 `링크:` URL로 마크다운 딥링크(`[제목](링크)`)를 걸어**
클릭 즉시 해당 이슈로 이동하게 한다. (제공된 링크만 쓰고 URL을 지어내지 않는다.)

* done 상태인데 완료조건이 다 체크되지 않은 이슈 (예: `[제목](링크) — done인데 완료조건 0/3`)
* [점검 대상 이슈]로 제공된 blocked 이슈
* [점검 대상 이슈]로 제공된 오래 정체된 in_progress 이슈
