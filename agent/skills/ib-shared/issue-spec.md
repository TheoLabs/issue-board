# 이슈 정규화 규격 (단일 원본)

기획을 **구현 가능한 이슈 트리**로 분해할 때의 정규 규격. `ib-issues`(단독)와 `ib-generate`
4단계, `ib-progress`가 이 규격을 공유한다.

## 트리 구조 (2단계)

- **에픽(부모 이슈, `type="epic"`)**: 기획서 "핵심 기능" 항목 하나당 하나가 기본.
- **작업 이슈(하위, `type="task"`)**: 각 에픽을 실제 착수 가능한 단위로 쪼갠다.
  규모 기준 하나당 **반나절~2일**. 너무 크면 더 쪼개고, 너무 잘면 합친다.

## 제목 — `[영역] 동사구`

- **영역**: 기능/도메인 축약 태그. 같은 영역끼리 묶이도록 **일관되게** 쓴다 (예: `Auth`,
  `Onboarding`, `Room`, `Race`, `Infra`). 한 프로젝트 안에서 표기를 통일하라(같은 대상에
  `Auth`/`인증`을 섞지 말 것).
- **동사구**: 무엇을 하는지 간결하게 (예: `소셜 로그인`, `방 생성`, `실시간 순위 계산`).
- 에픽이라고 `[에픽]`을 붙이지 마라 — 에픽/태스크 구분은 `type`이 담당한다.

## body — 에픽 (`type="epic"`)

```
{한 줄 요약 — 이 에픽이 달성하는 것}

## 범위
- {포함하는 하위 작업/영역}

## 완료 조건
- [ ] {에픽 수준에서 검증 가능한 조건}
```

## body — 태스크 (`type="task"`)

```
{한 줄 요약 — 무엇을 만들/바꾸는가}

## 작업 내용
- {구체적인 작업 항목}

## 완료 조건
- [ ] {검증 가능한 acceptance criteria}
```

## 공통 규칙

- 🔴 **완료 조건은 반드시 `- [ ]` 체크박스** — 이유·상세는 [conventions.md](conventions.md).
- 섹션 헤딩과 순서를 위 템플릿 그대로 지킨다. 첫 줄은 항상 한 줄 요약(헤딩 없이).
- **연관(기획·화면·도메인)은 body에 글로 적지 말고, 아래 "연동"의 구조적 링크로만 연결한다.**
  대시보드가 "기획 보기/화면 보기/도메인 보기" 버튼으로 렌더하므로, body에 중복하면 staleness만 생긴다.
- **`value`(가치)·`effort`(노력)로 평가** — **우선순위는 이 둘에서 자동 산출**되므로 `priority`를
  직접 넣지 마라.
  - `value`: 사용자/제품 가치. 핵심 유스케이스에 필수면 `high`, 보조면 `medium`, nice-to-have면 `low`.
  - `effort`: 구현 노력/복잡도. 크면 `high`, 보통 `medium`, 작으면 `low`.
  - 산출 규칙(참고): 高가치·低노력=높음, 高가치·高노력=보통, 低가치·高노력=낮음.
- `labels`: 영역 태그(예: `["backend"]`, `["ui"]`, `["infra"]`) — 과하지 않게. (이슈 키
  `CH-12`와는 별개 개념 — 키는 자동 부여, 라벨은 자유 태그.)

## 연동 (구조적 링크)

각 이슈를 만들 때 파생 출처를 **구조적으로 연결**한다:

- `planId`: 파생된 기획의 id. 보통 모든 이슈가 같은 기획을 가리킨다.
- `screenId`: 이슈가 다루는 화면이 있으면 와이어프레임의 해당 `data-screen` id (예: `race-detail`).
  UI와 무관한 이슈(인프라·백엔드)면 생략.
- `domainId`: 이슈가 특정 도메인(엔티티/테이블)을 다루면 그 도메인 id. 무관하면 생략.
- `applicationId`: 이슈가 속한 앱 id — **필수**. 자세한 규칙과 **이슈 키(`CH-12`)**는
  [application-model.md](application-model.md).

## 적재 순서 (중요)

1. **에픽을 먼저 생성**: `create_issue(projectId, title, body, type="epic", value, effort,
   labels, planId, screenId?, domainId?, applicationId)`. 반환된 각 에픽의 `id`·`key`를 기억한다.
2. **하위 이슈**를 생성하며 `type="task"`, `parentId`(또는 `parentKey`)에 해당 에픽을 넣고
   `planId`·`screenId`·`domainId`·`applicationId`를 채운다.
