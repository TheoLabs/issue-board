너는 이 프로젝트의 개발자다. 이슈보드에 쌓인 **오늘의 변경 이력(활동 로그)** 을 읽어
**정규화된 일일 업무 요약**을 만들고, **구글 드라이브에 업로드**한다.

요약의 원천은 이슈보드 서버의 활동 로그(ActivityLog)다. 추측하지 말고 **활동 로그에 실제로
기록된 것만** 요약한다.

## 대상 날짜

$ARGUMENTS

(비어 있으면: 오늘 — 서버 타임존(기본 Asia/Seoul) 기준.)

## 진행 순서

### 1) 프로젝트 · 활동 수집

- `get_project_context(repoPath=<cwd 절대경로>)`로 현재 프로젝트를 매칭한다(실패면 `list_projects`로 묻는다).
- `get_daily_activity(projectId, date=<대상 날짜|생략>, timezone="Asia/Seoul")`로 하루치 활동을 받는다.
  응답: `total`·`byEntity`·`byAction`·`bySource`·`activities[]`(`entityType`·`action`·`title`·
  `changes`·`source`·`createdAt`).
- `total`이 0이면: "오늘 기록된 변경이 없습니다"만 알리고 **업로드하지 않는다**(빈 문서 금지).

### 2) 표준 보고서로 요약 작성

**보고서의 페르소나·규칙·템플릿은 [ib-shared/daily-report.md](../ib-shared/daily-report.md) 하나에
정의**되어 있다. 그 파일을 읽어 **명세 그대로** 작성한다. (앱 내부 'AI 요약하기' 버튼/cron과
동일한 포맷을 공유한다.)

- 진척도·우선순위·분류는 이슈의 **현재 상태**가 필요하다. 활동만으로 부족하면 이슈 조회 도구로
  현재 status/priority/labels를 보강한다.
- 명세의 "채울 수 없는 항목은 넣지 않는다" 규칙을 지킨다.

### 3) 구글 드라이브 업로드

구글 드라이브 MCP 도구로 올린다. **폴더 = 루트/프로젝트, 파일 = 날짜별 문서.**

1. 루트 폴더 `이슈보드 일일요약`을 검색해 없으면 만든다.
2. 그 아래 프로젝트 폴더 `{프로젝트명}`을 검색해 없으면 만든다.
3. 그 폴더에 문서를 만든다 — 파일명 `{YYYY-MM-DD} {프로젝트명} 일일요약`, 내용은 2)의 마크다운
   (Google Docs 문서). **같은 날짜 문서가 있으면 새로 만들지 말고 갱신**(덮어쓰기).
4. 끝나면 **문서 링크(webViewLink)**를 보고한다.

> 구글 드라이브 연동이 없거나 실패하면: 요약 본문을 대화에 그대로 출력하고 업로드는 건너뛰었다고 알려라.

### 4) 보고

만든 요약의 핵심(총 건수 + TL;DR)과 **구글 드라이브 문서 링크**를 알린다.

## 매일 자동 실행 (스케줄)

무인으로 매일 돌리려면 이 스킬을 스케줄에 건다. 예:

```
/schedule 매일 19:00 KST 에 /ib-daily 를 이 프로젝트에서 실행해 구글 드라이브에 올려줘
```

스케줄 실행 환경에는 issue-board MCP와 구글 드라이브 연동이 모두 붙어 있어야 한다.

## 주의

- **활동 로그에 있는 것만** 요약한다. 코드·대화 기억으로 항목을 지어내지 마라.
- 활동 로그는 이 기능 도입 시점부터 쌓인다. 그 이전 날짜는 데이터가 없을 수 있다.
- 업로드는 되돌리기 번거로우니, 같은 날짜 문서는 **새로 만들지 말고 갱신**한다.
