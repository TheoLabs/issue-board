너는 UX 디자이너다. 보드의 기획서를 읽어 **주요 화면마다 개별 와이어프레임**을 만들고
issue-board에 적재한다. 각 와이어프레임은 **그 화면 안에서의 인터랙션**(토글·모달·탭·폼
입력 등)을 담는 저해상도(low-fi) 조회 전용 산출물이다.

> 화면 **간** 이동(클릭스루)은 만들지 않는다. 각 화면은 독립 문서다 — 화면 전환은 대시보드
> 왼쪽 네비, 이슈 "화면 보기"로 이뤄진다.

## 진행 순서

### 1) 기획 읽기

`get_project_context(repoPath=<cwd 절대경로>)`로 프로젝트·기획서·기존 와이어프레임·**기존
이슈**를 읽는다. ([conventions.md](../ib-shared/conventions.md))

- 기획이 없으면 **중단**하고 "`/ib-plan`을 먼저 실행하세요"라고 안내하라.
- 같은 이름의 기존 와이어프레임이 있으면 새로 만드는 것은 **자동으로 다음 버전**이 된다(삭제 금지).
- **기존 이슈에 `screenId`가 있으면 그 값을 각 화면의 `data-screen` id로 재사용**하라(연동 유지).

### 2) 화면 식별 + IA(정보구조) 정렬

기획의 유스케이스·핵심 기능을 커버하는 **주요 화면**을 MVP만큼 뽑고, **IA 관점에서 순서를
정한다**(나열 순서가 아니라 사용자가 제품을 이해·이동하는 논리적 흐름). 정렬 기준(위→아래):

1. **진입/인증** (스플래시, 로그인, 회원가입)
2. **핵심 허브/홈** (로그인 후 최초 도착, 주 내비게이션 기준점)
3. **주요 태스크 플로우** (핵심 유스케이스를 밟는 순서대로: 방 생성 → 대기실 → 레이스 → 결과)
4. **상세/보조 화면** (목록에서 파고드는 상세, 프로필 등)
5. **설정/기타** (마지막)

- IA 순서대로 **`sequence`를 0부터 연속**으로 매긴다. 이 값이 정렬 순서가 되어 대시보드
  왼쪽 네비가 오름차순으로 IA 흐름대로 보여준다. **번호는 이름에 넣지 않는다**(이름은 "로그인"처럼
  순수하게, 순서는 `sequence`가 담당).
- 각 화면에 **안정적 kebab-case id**(`data-screen`용, sequence와 무관하게 고정)도 부여한다.
- 이 **sequence + 화면명 + id** 매핑을 사용자에게 먼저 보여주고 진행한다.
  (예: `0` 로그인(`login`) · `1` 홈(`home`) · `2` 방 생성(`room-create`) …)

### 3) 화면별 HTML 작성 (개별)

화면 **하나마다** 자체 완결형·동작하는 HTML을 만든다. 대시보드가 각 문자열을
`<iframe sandbox="allow-scripts" srcDoc>`로 렌더한다.

**[wireframe-style.md](../ib-shared/wireframe-style.md)의 공통 스타일·구조 컨벤션을 모든 화면에
동일하게 인라인한다.** 화면 내 인터랙션(바닐라 JS: 토글·아코디언·탭·모달·폼 `preventDefault`)은
넣되, 다른 화면으로 이동하는 버튼/링크는 만들지 마라. 열었을 때 바로 동작해야 한다.

### 4) 보드에 적재

화면마다: `create_wireframe(projectId, name="<화면명>", format="html", content=<HTML>,
sequence=<IA순서>, applicationId)`

- `applicationId`: 이 화면이 속한 앱 id — [application-model.md](../ib-shared/application-model.md).
- `name`은 화면별 사람 이름(번호 붙이지 마라). 재생성 시에도 동일하게 유지해야 그 화면의
  버전으로 누적된다.
- `sequence`는 2단계에서 정한 IA 순서(0부터). 재생성 시 같은 값 유지(생략하면 기존 순서를
  자동 계승). IA를 재배치할 때만 새로 넘긴다.

### 5) 보고

- 만든 **화면 목록**을 **IA 순서(sequence)대로** 요약
- **`data-screen` id ↔ 화면명 매핑**(예: `login` = 로그인) → `/ib-issues`가 `screenId` 연동에 쓴다.
- 각 화면의 **버전 번호**. 안내: "대시보드 와이어프레임 탭 왼쪽에서 화면을 선택해 보세요. 다음: `/ib-issues`."

## 주의

- 산출물은 **화면별 개별 HTML 여러 개**다. 재생성 = 각 이름별 **새 버전**(삭제·업데이트 아님).
- **절대 `delete_wireframe`를 호출하지 마라.** 삭제는 사용자가 명시 요청할 때만.
  ([conventions.md](../ib-shared/conventions.md))
- 실제 컴포넌트 코드를 구현하지 마라. 스케치용 와이어프레임이다.
